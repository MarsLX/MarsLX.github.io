20200423靶机搜索不到ip的问题研究

发表于 2020-04-23

之前一直困扰的问题，今下午终于研究解决了。

问题描述：从vulnhub下载靶机，导入虚拟机后，dhcp不启动，无法对ip进行探测。

解决办法：

0x01

在进入系统界面的时候，按e，找到有 ro XXXXXX quiet的一行，把原本的内容删除，并且改写成rw single init=/bin/bash，然后按ctrl+x，进入单用户模式。

0x02

使用命令ip a查看当前使用了哪些端口

在对dc-1～dc-9系列靶机进行导入测试时，遇到两种情况，一种是eth0，一种是ens33，

不管哪一种，只需要在下一步键入对应的端口即可。

0x03

vi /etc/network/interfaces

打开后根据刚才ip a查找到的端口，输入，要保证有两条命令

auto eth0

iface eth0 inet dhcp

如果是ens33就如图

注意vi在输入时，o是在当前光标位置的下一个位置输入。退出时记得保存。

0x04

/etc/init.d/networking start

会自动执行dhcp并分配ip

使用ifconfig即可查看到ip

(其实这一步不执行也ok，这一步是为了验证刚才执行结果有效，因为当前还是在单用户模式下)

记得重启正常进入linux
